KEY DIFFERENCES FROM KINETIC:
  • No borders (maximum screen space)
  • Single-line status (●ready model·3)
  • Transcript format (no bubbles)
  • Raw markers: → tool, ← ✓ success, ← ✗ error
  • Brutalist, raw terminal aesthetic

┌────────────────────────────────────────────────────────────────────────────────┐
│ DESIGN PRINCIPLES (Kinetic TUI)                                                  │
└────────────────────────────────────────────────────────────────────────────────┘

1. Information First      → Every pixel conveys state
2. Motion = Life          → Static feels dead, kinetic feels alive
3. Density Over Whitespace→ Maximize information per line
4. Symbol Over Label      → Progress bar > "45% used"
5. Glitch Aesthetic       → Embrace raw terminal, don't hide it

│ SUMMARY                                                                         │
└────────────────────────────────────────────────────────────────────────────────┘

The current Ferroclaw TUI (Kinetic V3) features:
  • 2-line kinetic status with progress bar and token velocity
  • Animated thinking indicator (● runs, ○ ready)
  • Glitch effects on tool calls (⚡ flicker for 2 frames)
  • Glitter verbs for contextual status updates
  • Hermes-style borders around chat and input
  • Chat bubble format for messages
  • Color-coded tool results (green ✓, red ✗)

This diagram serves as a visual reference for the current TUI layout
to guide the redesign process.
