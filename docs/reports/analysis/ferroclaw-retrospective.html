MCP: Native Integration

Model Context Protocol built into the core.

  • Zero external deps — MCP client is pure Rust, vendored
  • Dynamic tool loading — Connect to MCP servers, get tools instantly
  • Automatic compression — dietmcp reduces context by 40%
  • Sandboxed execution — Each tool runs in isolated scope
// Loading tools from an MCP server

  agent.register_tool(tool); // instantly available
}

07

Security: Not an Afterthought

Every security decision is explicit and escalated.

  • Audit logging — Every tool call, every token, logged to tamper-evident store
  • No hardcoded secrets — Runtime validation, fail fast if missing
  • Escalation protocol — Auth/crypto code requires human review before shipping
  • Sandboxed tools — Each tool runs in isolated scope with timeout

Result: You can review everything the agent did. In production. Right now.

Challenge: Chat History Scrolling

Users couldn't scroll. The conversation grew unreadable.

// The bug: inverted scroll logic
fn scroll_offset(total: u16, visible: u16, offset: u16) -> u16 {
  // WRONG: shows oldest when offset = 0
  max_scroll.saturating_sub(offset) // ❌

  // RIGHT: shows newest when offset = 0
  offset // ✅
}

Fixed the math. Now PageUp/PageDown work like a normal chat. Mouse wheel too.

Challenge: The 'c' Key Hijack

User: "I can't type the letter c!"

// The orchestrator TUI had task shortcuts

  toggle_task_status(); // ❌ blocked typing!

  • Task list feature was cool but broke basic typing
  • Disabled task shortcuts, kept 'n' and 'd' blocked
  • Minimal/kinetic TUIs removed task list entirely
  • Lesson: Features can't break fundamentals
14

Challenge: 28 Build Errors

Sync vs async. Wrong mutex types. Missing imports.

// Error: bool is not a future
let has_node = node_modules.exists().await; // ❌

// Fix: exists() is synchronous
let has_node = node_modules.exists(); // ✅

// Error: wrong mutex for async
std::sync::Mutex // ❌ blocks async runtime
tokio::sync::Mutex // ✅ async-friendly

One by one. Fixed them all. Zero warnings remains.
